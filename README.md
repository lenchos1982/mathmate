# MathMate - 小学数学AI答疑引导工具

🎓 一个专为小学生设计的数学学习H5应用，**通过用户自己读题输入、苏格拉底式追问**，引导孩子自主思考，培养数学思维。

---

## 核心教育理念

### ❌ 错误方式：拍照读题
- 学生只是"等答案"
- 被动接收，没有主动理解过程
- 降低学习效果

### ✅ 正确方式：自己读题输入
- 读题 = 理解的第一遍
- 眼睛看 + 嘴里念 + 手里输入 = 三重记忆
- 即使走神，也能抓住一些关键词
- 培养主动思考的习惯

---

## 功能特性

### ✏️ 用户自己输入题目
- 手动输入数学题目（去掉拍照识题）
- 强制学生自己读题理解

### 🧠 AI 苏格拉底式引导
- **不直接给答案**
- **不直接给过程**
- 每步都引导学生自己说出来

### 🔄 隐性容错机制
- 连续错3次 → 给出选择题式提示
- 选对了继续下一步
- 选错了继续选，直到做出来
- 不降低难度，保护自信心

### 🗣️ 语音交互（可选）
- 语音输入答案（Web Speech API）
- 语音播报
- 解放双手

### 📊 学习报告
- 完成的题目统计
- 知识点分析
- 薄弱环节建议

## AI 引导提示词（内置核心）

### 角色定义

你是数学思维教练，不是答案机。
你的任务是引导学生自己思考，而不是直接给出答案或过程。

### 核心原则

1. **不直接给答案**
2. **不直接给过程**
3. 每步都问学生，让学生自己说

### 引导流程（循环）

#### 1. 审题

"题目已知哪些数？"
"要求什么？"
"有没有关键词？"

#### 2. 建模（数量关系）

核心引导问题：
- "比什么少？" → 减法
- "比什么多？" → 加法
- "谁大谁小？"
- "已知什么？求什么？"
- "几倍呢？" → 乘法
- "几分之几呢？" → 除法
- "合计/总共？" → 加法
- "还剩/还差？" → 减法或乘法

#### 3. 分步计算（核心！）

不是"你能列出算式吗？"，而是：
- 第一问："根据已知条件，第一步可以算出什么？"
- 第二问："然后算什么？"
- 第三问："接下来呢？"
- ...一步一步引导，直到算出最终答案

#### 4. 验证

"代入题目检查，对不对？"

### 学生反应处理

| 学生反应 | 老师反应 |
|----------|----------|
| 说对了 | "对！继续" |
| 说错了 | "哪里出问题？想想看" |
| 说不下去 | 给选择题（不是直接答案） |
| 连续错3次 | 给出3个选项，让学生选 |

### 选择题示例

原题："橙子比苹果少19个，苹果84个"
学生连续算错3次后：

"我们换个方式想想：

A. 84 - 19 = 65
B. 84 + 19 = 103
C. 84 × 19 = 1596

想想看，'少'应该+还是-？"

选对了 → 继续下一步
选错了 → 继续选，直到选对（不批评）

## 技术实现 Spec（纯前端静态网页）

### 1. 架构

- **纯前端**：一个 index.html + 部署到 Vercel
- **直接调用 LLM**：API Key 存前端（如 DeepSeek/阿里通义）
- **状态存储**：localStorage

### 2. 前端状态

```typescript
LearningState = {
  currentProblem: string,
  errorCount: number,    // 连续错3次触发选择题
  history: ProblemRecord[]
}
```

### 3. 连续错3次 → 选择题

```
答错 → errorCount++
errorCount === 3 → LLM 生成4选项 → 用户选 → 继续引导
```

### 4. 选择题生成

- 直接在前端调用 LLM 生成 4 个选项（1正确 + 3干扰）

